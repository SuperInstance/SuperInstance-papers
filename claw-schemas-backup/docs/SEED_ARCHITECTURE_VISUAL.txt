CLAW SEED FRAMEWORK - ARCHITECTURE
====================================

                          SEED LIFECYCLE
                          =============

  [CREATED]       [TRAINING]        [STABILIZED]       [DISTILLED]      [DEPLOYED]
     |               |                  |                  |                |
     v               v                  v                  v                v
  [Define]       [Train]           [Validate]         [Compress]       [Execute]
  [Seed]         [Params]          [Metrics]          [Model]          [inProd]


                       SEED STRUCTURE
                       =============

  +--------------------------------------------------------------------------+
  |                              SEED                                         |
  +--------------------------------------------------------------------------+
  |                                                                          |
  |  [PURPOSE]    [TRIGGER]    [STRATEGY]                                   |
  |  "What I      "When I      "How I                                       |
  |   do"         activate"    learn"                                       |
  |                                                                          |
  |  [EQUIPMENT]  [TRAINING]   [LEARNED]                                    |
  |  "Tools I     "Data I      [PARAMETERS]                                 |
  |   use"        learn from"   "What I learned"                            |
  |                                                                          |
  |  [MEMORY]     [SOURCE]     [WEIGHTS]                                    |
  |  [REASONING]  [ITERATIONS] [THRESHOLDS]                                 |
  |  [CONSENSUS]  [RATE]       [METRICS]                                    |
  |  [SPREADSHEET][BATCH]                                                   |
  |  [DISTILL]                                                              |
  |  [COORD]                                                                |
  |                                                                          |
  +--------------------------------------------------------------------------+


                    TRIGGER TYPES
                    ============

  [PERIODIC]                    [DATA-DRIVEN]                [EVENT-BASED]
  "Every 5 min"                "When CPU > 80%"             "On error.*"

  Schedule: PT5M               Field: cpu                   Pattern: error.*
  Phase:   PT0S                Operator: >                  Filter: critical
  Jitter:  PT30S               Value: 0.8
                                Window: PT2M

                        [HYBRID]
                        "Combine triggers with AND/OR/NOT"

                        Ex: Every hour OR On emergency


                  LEARNING STRATEGIES
                  ==================

  [REINFORCEMENT]     [SUPERVISED]        [UNSUPERVISED]     [FEW-SHOT]
  Trial & error       Labeled data        Pattern discovery  Learn from few
  with rewards        with teacher        no labels          examples

  Reward: fn          Labels: human      Pattern: cluster   Examples: [...]
  Explore: 0.1        Quality: high      Type: anomaly      Meta-rate: 0.01
  Discount: 0.99

  Optimization        Classification      Discovery          Prototyping
  Control             Prediction          Clustering         Low-data


                    EQUIPMENT SYSTEM
                    ===============

  [MEMORY]      [REASONING]    [CONSENSUS]     [SPREADSHEET]
  Storage       Inference      Voting          Analytics
  Retrieval     Planning       Agreement       Reporting
  Caching       Analysis       Coordination    Transform

  [DISTILLATION]              [COORDINATION]
  Compression                 Orchestration
  Optimization                Workflow
  Deployment                  Task distribution

                    COMMON PATTERNS
                    ==============

  PATTERN 1: Monitor-Optimize
  Trigger: Periodic (PT5M)
  Strategy: Reinforcement
  Equipment: MEMORY, REASONING

  PATTERN 2: Event-Response
  Trigger: Event-Based (error.*)
  Strategy: Few-Shot
  Equipment: COORDINATION

  PATTERN 3: Threshold Alert
  Trigger: Data-Driven (cpu > 0.9)
  Strategy: Supervised
  Equipment: SPREADSHEET

  PATTERN 4: Adaptive Control
  Trigger: Hybrid (schedule OR emergency)
  Strategy: Reinforcement
  Equipment: MEMORY, REASONING, COORDINATION

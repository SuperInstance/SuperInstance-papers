## QUICK REFERENCE FOR SUCCESSORS

### Mathematical Clarity Checklist
- [ ] Introduce concept before notation
- [ ] Provide concrete example before abstract statement
- [ ] Explain "why this matters" before "what this is"
- [ ] Include visual intuition before formal proof
- [ ] Show failure cases before claiming success
- [ ] Connect to reader's experience before novel terminology

### Common Mathematical Pitfalls to Avoid
1. **The Orphan Equation**: Equation appears without textual explanation
2. **The Notation Bomb**: Symbols appear without definition key
3. **The Proof Drop**: Formal proof without intuitive walkthrough
4. **The Assumption Trap**: Implicit prerequisites not stated
5. **The Triviality Fallacy": Assuming "clearly" is ever clear

### Key Insight
Mathematical clarity isn't about dumbing down - it's about building up understanding systematically. Every formal element should reward the reader's investment of attention.

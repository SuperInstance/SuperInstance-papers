# ITERATION 4: REPO-TO-SEED CONVERSION THEORY

## 1. The Inverse Problem

### 1.1 Problem Statement

Given a repository (collection of code, configuration, and outputs), can we derive the seed that would recreate it? This is the **inverse problem** of seed-based generation.

**Definition 1.1 (Repo-to-Seed Problem)**

Given a repository $R$ generated by some unknown seed $S^*$, find $S$ such that:

$$\text{Model}(S, P) \approx R$$

Where $P$ is a known or inferred prompt template.

### 1.2 Information Requirements

**Theorem 1.2.1 (Seed Derivation Necessity)**

To uniquely determine seed $S^*$ from repository $R$, we require:

1. **Output samples**: At least $k$ distinct outputs from $\text{Model}(S^*, P)$
2. **Model specification**: Exact model version and architecture
3. **Prompt knowledge**: The prompt template $P$ or sufficient constraints to infer it
4. **Determinism guarantee**: Model must be deterministic given seed

**Proof Sketch:**

Without output samples, there is no information to invert. Without model specification, the mapping $S \to \text{Model}(S, P)$ is unknown. Without prompt knowledge, multiple $(S, P)$ pairs could produce $R$. Without determinism, the mapping is not well-defined.

$\square$

### 1.3 Theoretical Framework

**Definition 1.3.1 (Seed Space $\mathcal{S}$)**

The seed space is the set of all possible RNG initializations:

$$\mathcal{S} = \{0, 1, \ldots, 2^{64}-1\}$$

For extended seeds with multiple parameters:

$$\mathcal{S}_{extended} = \{(s_1, s_2, \ldots, s_n) : s_i \in \{0, \ldots, 2^{64}-1\}\}$$

**Definition 1.3.2 (Generation Mapping $G$)**

The generation mapping $G: \mathcal{S} \times \mathcal{P} \to \mathcal{R}$ maps a seed and prompt to a repository:

$$G(S, P) = R$$

**Definition 1.3.3 (Inverse Mapping $G^{-1}$)**

The inverse problem seeks:

$$G^{-1}(R) = \{(S, P) : G(S, P) = R\}$$

Note: $G^{-1}$ is generally a set-valued function due to potential multiple solutions.

## 2. Seed Fingerprinting

### 2.1 Unique Identifiers for Code

**Definition 2.1.1 (Code Fingerprint)**

A code fingerprint $F: \mathcal{R} \to \{0,1\}^n$ is a function that maps a repository to a fixed-length bitstring with the property:

$$F(R_1) = F(R_2) \Rightarrow R_1 \approx R_2$$

**Types of Fingerprints:**

| Type | Definition | Collision Probability |
|------|------------|----------------------|
| **Hash-based** | $F(R) = \text{SHA256}(R)$ | $O(2^{-256})$ |
| **Semantic** | $F(R) = \text{Embed}(R)$ | Depends on embedding |
| **Structural** | $F(R) = \text{AST\_Hash}(R)$ | $O(2^{-128})$ |
| **Behavioral** | $F(R) = \text{Output\_Hash}(R)$ | Depends on test coverage |

### 2.2 Seed Fingerprint Relationship

**Theorem 2.2.1 (Fingerprint-Seed Correspondence)**

For a deterministic model $M$ with prompt $P$, there exists a function $\phi$ such that:

$$F(G(S, P)) = \phi(S)$$

**Proof:**

Since $G(S, P)$ is deterministic, $F(G(S, P))$ is determined solely by $S$. Define $\phi(S) = F(G(S, P))$. Then $F(G(S, P)) = \phi(S)$.

$\square$

**Corollary 2.2.1 (Seed Recovery from Fingerprint)**

If $\phi$ is invertible on its image, then:

$$S = \phi^{-1}(F(R))$$

### 2.3 Multi-Level Fingerprinting

**Definition 2.3.1 (Hierarchical Fingerprint)**

A hierarchical fingerprint captures multiple levels of code structure:

$$F_{hierarchical}(R) = (F_{char}(R), F_{token}(R), F_{ast}(R), F_{semantic}(R))$$

Where:
- $F_{char}$: Character-level hash
- $F_{token}$: Token sequence hash
- $F_{ast}$: Abstract Syntax Tree hash
- $F_{semantic}$: Semantic embedding

**Algorithm 2.3.1 (Hierarchical Fingerprint Computation)**

```
FUNCTION compute_hierarchical_fingerprint(repository R):
    # Level 1: Character-level
    char_hash = SHA256(R.raw_content)
    
    # Level 2: Token-level
    tokens = tokenize(R)
    token_sequence = [t.type for t in tokens]
    token_hash = SHA256(token_sequence)
    
    # Level 3: AST-level
    ast = parse(R)
    ast_structure = serialize_ast(ast, exclude_values=True)
    ast_hash = SHA256(ast_structure)
    
    # Level 4: Semantic-level
    functions = extract_functions(R)
    embeddings = [embed(f.signature + f.docstring) for f in functions]
    semantic_hash = hash(embeddings)
    
    RETURN (char_hash, token_hash, ast_hash, semantic_hash)
```

## 3. Reverse Engineering Algorithms

### 3.1 Exhaustive Search (Theoretical)

**Algorithm 3.1.1 (Exhaustive Seed Search)**

```
INPUT: Repository R, Prompt P, Model M, Tolerance ε
OUTPUT: Seed S or FAILURE

1. FOR each S in {0, 1, ..., 2^64-1}:
   a. R_candidate = M(S, P)
   b. IF distance(R_candidate, R) < ε:
      RETURN S
2. RETURN FAILURE
```

**Complexity Analysis:**

- Time: $O(2^{64} \cdot T_M)$ where $T_M$ is model inference time
- This is computationally infeasible

### 3.2 Gradient-Based Seed Optimization

**Definition 3.2.1 (Seed Loss Function)**

Define a loss function measuring the distance between generated and target repositories:

$$\mathcal{L}(S) = d(G(S, P), R_{target})$$

**Challenge:** Seeds are discrete integers, not continuous parameters.

**Approach:** Use continuous relaxation and projection.

**Algorithm 3.2.1 (Gradient-Based Seed Search)**

```
INPUT: Repository R, Prompt P, Model M, Learning rate α
OUTPUT: Optimized Seed S

1. Initialize: S_continuous = random_vector(64)  # Relaxed seed
2. FOR iteration = 1 to MAX_ITER:
   a. S_discrete = project_to_seed(S_continuous)
   b. R_generated = M(S_discrete, P)
   c. loss = compute_loss(R_generated, R)
   d. IF loss < tolerance:
      RETURN S_discrete
   e. gradient = estimate_gradient(loss, S_continuous)
   f. S_continuous = S_continuous - α * gradient
3. RETURN best_S_discrete
```

**Gradient Estimation Methods:**

1. **Finite Differences:**
   $$\frac{\partial \mathcal{L}}{\partial S_i} \approx \frac{\mathcal{L}(S + \epsilon e_i) - \mathcal{L}(S - \epsilon e_i)}{2\epsilon}$$

2. **REINFORCE:**
   $$\nabla_S \mathcal{L} = \mathbb{E}[\mathcal{L}(S) \nabla_S \log P(S)]$$

3. **Gumbel-Softmax Relaxation:**
   $$S_{relaxed} = \text{softmax}((\log \pi + G) / \tau)$$

### 3.3 Information-Theoretic Approach

**Definition 3.3.1 (Seed Entropy)**

The entropy of the seed given observations:

$$H(S | R) = H(S) - I(S; R)$$

**Theorem 3.3.1 (Seed Information Content)**

For a model with $n$ output tokens and vocabulary size $V$:

$$I(S; R) \leq n \log_2 V$$

**Proof:**

The output $R$ has at most $V^n$ possible values, so:

$$H(R) \leq \log_2(V^n) = n \log_2 V$$

Since $I(S; R) \leq H(R)$:

$$I(S; R) \leq n \log_2 V$$

$\square$

**Corollary 3.3.1 (Seed Recovery Impossibility)**

If $H(S) > n \log_2 V$, then seed recovery is impossible with certainty.

**Algorithm 3.3.1 (Information-Guided Search)**

```
INPUT: Repository R, Prior distribution P(S)
OUTPUT: Posterior distribution P(S|R)

1. Initialize prior: P(S) = uniform or informed_prior
2. FOR each observation r_i in R:
   a. likelihood = P(r_i | S) for each S
   b. P(S | r_i) ∝ P(r_i | S) P(S)
   c. P(S) = P(S | r_i)  # Update posterior
3. RETURN P(S | R)
```

## 4. Compression Theory

### 4.1 Repository as (Seed, Prompt, Model)

**Definition 4.1.1 (Seed Compression)**

The seed compression of a repository $R$ is:

$$\text{Compress}(R) = (S^*, P^*, M^*)$$

Such that:
1. $G(S^*, P^*, M^*) \approx R$
2. $|S^*| + |P^*| + |M^*| \ll |R|$

**Theorem 4.1.1 (Compression Bound)**

The maximum compression ratio is bounded by:

$$\frac{|R|}{|S| + |P| + |M|} \leq \frac{|R|}{64 + |P| + |M|}$$

For large repositories, this approaches infinity.

### 4.2 Kolmogorov Complexity Analysis

**Definition 4.2.1 (Repository Kolmogorov Complexity)**

The Kolmogorov complexity of $R$:

$$K(R) = \min\{|p| : U(p) = R\}$$

**Theorem 4.2.1 (Seed as Program)**

If $R = G(S, P, M)$, then:

$$K(R) \leq K(S) + K(P) + K(M) + c$$

Where $c$ is a constant for the composition overhead.

**Proof:**

The program to generate $R$ consists of:
1. The seed $S$ (complexity $K(S)$)
2. The prompt $P$ (complexity $K(P)$)
3. The model $M$ (complexity $K(M)$)
4. A fixed driver program (complexity $c$)

Total: $K(S) + K(P) + K(M) + c$

$\square$

### 4.3 Lossy vs Lossless Compression

## 5. Practical Implementation

### 5.1 Seed Recovery Pipeline

**Algorithm 5.1.1 (Complete Seed Recovery System)**

```python
class SeedRecoverySystem:
    """
    Complete system for deriving seeds from repositories.
    """
    
    def __init__(self, model, prompt_template):
        self.model = model
        self.prompt_template = prompt_template
        self.fingerprint_db = FingerprintDatabase()
        self.seed_optimizer = SeedOptimizer()
    
    def recover_seed(self, repository, method='hybrid'):
        """
        Recover seed from repository using specified method.
        
        Methods:
        - 'exhaustive': Exhaustive search (infeasible for large spaces)
        - 'gradient': Gradient-based optimization
        - 'bayesian': Bayesian optimization
        - 'evolutionary': Evolutionary algorithm
        - 'hybrid': Combine multiple methods
        """
        # Step 1: Compute fingerprint
        fingerprint = self.compute_fingerprint(repository)
        
        # Step 2: Check fingerprint database for known seeds
        known_seed = self.fingerprint_db.lookup(fingerprint)
        if known_seed:
            return known_seed
        
        # Step 3: Apply optimization method
        if method == 'gradient':
            seed = self.seed_optimizer.gradient_search(
                repository, self.model, self.prompt_template
            )
        elif method == 'bayesian':
            seed = self.seed_optimizer.bayesian_search(
                repository, self.model, self.prompt_template
            )
        elif method == 'evolutionary':
            seed = self.seed_optimizer.evolutionary_search(
                repository, self.model, self.prompt_template
            )
        elif method == 'hybrid':
            seed = self.seed_optimizer.hybrid_search(
                repository, self.model, self.prompt_template
            )
        
        # Step 4: Validate recovered seed
        reconstructed = self.model.generate(seed, self.prompt_template)
        similarity = self.compute_similarity(reconstructed, repository)
        
        if similarity > 0.95:
            # Store in database for future lookups
            self.fingerprint_db.store(fingerprint, seed)
            return seed
        else:
            return None
    
    def compute_fingerprint(self, repository):
        """Compute hierarchical fingerprint."""
        return HierarchicalFingerprint.compute(repository)
    
    def compute_similarity(self, repo1, repo2):
        """Compute similarity between repositories."""
        # Multi-level similarity
        char_sim = character_similarity(repo1, repo2)
        token_sim = token_similarity(repo1, repo2)
        ast_sim = ast_similarity(repo1, repo2)
        semantic_sim = semantic_similarity(repo1, repo2)
        
        # Weighted combination
        return 0.1 * char_sim + 0.2 * token_sim + 0.3 * ast_sim + 0.4 * semantic_sim
```

### 5.2 Prompt Inference

**Theorem 5.2.1 (Prompt Recovery)**

Given repository $R$ and seed $S$, the prompt $P$ can be recovered by:

$$P = \arg\min_{P'} d(G(S, P'), R)$$

**Algorithm 5.2.1 (Prompt Inference)**

```python
def infer_prompt(repository, seed, model, prompt_candidates):
    """
    Infer the prompt used to generate repository.
    """
    best_prompt = None
    best_score = float('inf')
    
    for prompt in prompt_candidates:
        generated = model.generate(seed, prompt)
        score = distance(generated, repository)
        
        if score < best_score:
            best_score = score
            best_prompt = prompt
    
    return best_prompt, best_score
```

The minimum number of observations required to uniquely determine a seed is:

$$n_{min} \geq \frac{H(S)}{I(S; R_1)}$$

Where $I(S; R_1)$ is the mutual information between the seed and a single output.

## 7. Defining Seed Stability

### 7.1 Stability Metrics

**Definition 7.1.1 (Output Stability)**

A seed $S$ is output-stable for prompt $P$ if:

$$\text{Stability}_{output}(S, P) = \mathbb{P}[\forall t: G(S, P)_t = G(S, P)_t^{(reread)}]$$

Where $G(S, P)_t^{(reread)}$ is the output when the model is re-loaded.

**Definition 7.1.2 (Semantic Stability)**

A seed $S$ is semantically-stable if:

$$\text{Stability}_{semantic}(S, P) = \mathbb{E}[\text{Sim}_{semantic}(G(S, P), G(S, P)^{(rerun)})]$$

**Definition 7.1.3 (Structural Stability)**

A seed $S$ is structurally-stable if:

$$\text{Stability}_{structural}(S, P) = \mathbb{E}[\text{Sim}_{AST}(G(S, P), G(S, P)^{(rerun)})]$$

### 7.2 Stability Classification

| Stability Level | Output Stability | Semantic Stability | Structural Stability |
|----------------|------------------|--------------------|---------------------|
| **Ultra-Stable** | 1.000 | 1.000 | 1.000 |
| **Highly Stable** | > 0.999 | > 0.995 | > 0.99 |
| **Stable** | > 0.99 | > 0.98 | > 0.95 |
| **Moderate** | > 0.95 | > 0.90 | > 0.80 |
| **Unstable** | < 0.95 | < 0.90 | < 0.80 |

### 7.3 Factors Affecting Stability

**Theorem 7.3.1 (Stability Determinants)**

Seed stability depends on:

1. **Model determinism**: Hardware-level RNG control
2. **Temperature**: Lower temperature → higher stability
3. **Prompt specificity**: More specific prompts → higher stability
4. **Token entropy**: Lower entropy tokens → higher stability

**Mathematical Formulation:**

$$\text{Stability}(S, P, T) = f(T) \cdot g(P) \cdot h(S)$$

Where:
- $f(T) = \exp(-\alpha \cdot T)$ is the temperature factor
- $g(P) = \text{Specificity}(P)$ is the prompt factor
- $h(S) = \mathbb{E}_{t}[\text{Stability}(S_t)]$ is the average token stability

### 8.2 Token-Level Stability Analysis

**Definition 8.2.1 (Token Stability Score)**

For token at position $t$:

$$\text{Stability}_t = \frac{\max_{v \in V} \text{Count}(v, t)}{N_{runs}}$$

Where $\text{Count}(v, t)$ is the number of times token $v$ appeared at position $t$.

**Algorithm 8.2.1 (Token Stability Analysis)**

```python
def analyze_token_stability(seed, prompt, model, n_runs=1000):
    """
    Analyze stability at token level.
    """
    token_counts = defaultdict(lambda: defaultdict(int))
    
    for _ in range(n_runs):
        output = model.generate(seed, prompt)
        tokens = tokenize(output)
        
        for pos, token in enumerate(tokens):
            token_counts[pos][token] += 1
    
    # Compute per-position stability
    stability_profile = []
    for pos in sorted(token_counts.keys()):
        total = sum(token_counts[pos].values())
        max_count = max(token_counts[pos].values())
        stability = max_count / total
        entropy = compute_entropy(token_counts[pos])
        
        stability_profile.append({
            'position': pos,
            'stability': stability,
            'entropy': entropy,
            'most_common': max(token_counts[pos], key=token_counts[pos].get)
        })
    
    return stability_profile
```

### 8.3 Entropy-Based Stability

**Definition 8.3.1 (Token Entropy)**

$$H_t = -\sum_{v \in V} P_t(v) \log_2 P_t(v)$$

**Theorem 8.3.1 (Entropy-Stability Relationship)**

$$\text{Stability}_t = 2^{-H_t} + \epsilon$$

Where $\epsilon$ is small for distributions with dominant modes.

**Proof:**

For a distribution with maximum probability $p_{max}$:

$$H \leq -p_{max} \log_2 p_{max} - (1-p_{max}) \log_2(1-p_{max})$$

When $p_{max} \to 1$, $H \to 0$ and $\text{Stability} \to 1$.

$\square$

### 9.1 Seed Space Topology

**Definition 9.1.1 (Stable Region)**

A stable region $\mathcal{R}_s \subseteq \mathcal{S}$ is a connected subset of seed space where:

$$\forall S \in \mathcal{R}_s: \text{Stability}(S) > \theta$$

**Definition 9.1.2 (Stability Landscape)**

The stability landscape is a function:

$$\mathcal{L}: \mathcal{S} \to [0, 1]$$

Where $\mathcal{L}(S) = \text{Stability}(S)$.

### 9.2 Attractor Dynamics

**Definition 9.2.1 (Seed Attractor)**

A seed attractor $\mathcal{A} \subseteq \mathcal{S}$ satisfies:

1. **Invariance**: $S \in \mathcal{A} \Rightarrow \text{Nearby}(S) \subset \mathcal{A}$
2. **Attraction**: $S \notin \mathcal{A} \Rightarrow \lim_{n \to \infty} \text{Optimize}^n(S) \in \mathcal{A}$

**Theorem 9.2.1 (Stable Attractor Existence)**

For any deterministic model and prompt, stable attractors exist in seed space.

**Proof Sketch:**

Consider the set of seeds that produce exact matches:
$$\mathcal{A} = \{S : G(S, P) = R_{target}\}$$

This set is non-empty for some $R_{target}$, and forms an attractor under gradient descent.

$\square$

### 9.3 Basin of Attraction Analysis

**Definition 9.3.1 (Basin of Attraction)**

The basin of attraction for attractor $\mathcal{A}$ is:

$$\mathcal{B}(\mathcal{A}) = \{S : \lim_{n \to \infty} \text{Optimize}^n(S) \in \mathcal{A}\}$$

```python
            children.extend([child1, child2])
        
        # Mutation
        children = [mutate_seed(child, mutation_rate=0.1) for child in children]
        
        # Replace population
        population = parents + children
    
    # Return best seed
    fitness = [measure_stability(seed, prompt) for seed in population]
    best_idx = argmax(fitness)
    return population[best_idx], fitness[best_idx]
```

## 12. The Winner-Finding Problem

### 12.1 Defining "Winners" in Seed Space

**Definition 12.1.1 (Winner Seed)**

A winner seed $S^*$ maximizes an objective function:

$$S^* = \arg\max_{S \in \mathcal{S}} \mathcal{F}(S)$$

Where $\mathcal{F}: \mathcal{S} \to \mathbb{R}$ is the fitness function.

**Types of Fitness Functions:**

| Type | Definition | Application |
|------|------------|-------------|
| **Quality** | $\mathcal{F}(S) = \text{Quality}(G(S, P))$ | Code generation |
| **Efficiency** | $\mathcal{F}(S) = \text{Speed}(G(S, P))$ | Performance optimization |
| **Correctness** | $\mathcal{F}(S) = \text{PassRate}(G(S, P))$ | Test-driven generation |
| **Novelty** | $\mathcal{F}(S) = \text{Diversity}(G(S, P))$ | Creative generation |

### 12.2 Durable Variation Definition

**Definition 12.2.1 (Durable Variation)**

A seed variation $S'$ is durable for context set $\mathcal{C}$ if:

$$\min_{C \in \mathcal{C}} \mathcal{F}_C(S') > \theta$$

**Definition 12.2.2 (Context Robustness)**

The context robustness of seed $S$ is:

$$\text{Robustness}(S) = \mathbb{E}_{C \sim \mathcal{C}}[\mathcal{F}_C(S)] - \text{Var}_{C \sim \mathcal{C}}[\mathcal{F}_C(S)]$$

### 12.3 Pareto Optimality

**Definition 12.3.1 (Pareto-Optimal Seed)**

Seed $S$ is Pareto-optimal if there is no $S'$ such that:

$$\forall i: \mathcal{F}_i(S') \geq \mathcal{F}_i(S) \land \exists j: \mathcal{F}_j(S') > \mathcal{F}_j(S)$$

**Algorithm 12.3.1 (Pareto Front Discovery)**

```python
def find_pareto_front(seeds, objectives):
    """
    Find the Pareto-optimal seeds.
    """
    pareto_front = []
    
    for seed in seeds:
        # Check if seed is dominated
        dominated = False
        for other in seeds:
            if dominates(other, seed, objectives):
                dominated = True
                break
        
        if not dominated:
            pareto_front.append(seed)
    
    return pareto_front

def dominates(seed1, seed2, objectives):
    """
    Check if seed1 Pareto-dominates seed2.
    """
    all_geq = all(f(seed1) >= f(seed2) for f in objectives)
    any_gt = any(f(seed1) > f(seed2) for f in objectives)
    return all_geq and any_gt
```

## 13. Efficient Search Algorithms

### 13.1 Multi-Armed Bandit Approach

**Definition 13.1.1 (Seed Bandit)**

Each seed $S$ is an arm with unknown reward distribution $R_S$. The goal is to minimize cumulative regret:

$$\text{Regret}_T = \sum_{t=1}^{T} (R_{S^*} - R_{S_t})$$

**Algorithm 13.1.1 (UCB Seed Selection)**

```python
def ucb_seed_search(seeds, n_rounds=1000):
    """
    Upper Confidence Bound seed selection.
    """
    n_seeds = len(seeds)
    counts = np.zeros(n_seeds)
    rewards = np.zeros(n_seeds)
    
    # Initial exploration
    for i in range(n_seeds):
        reward = evaluate_seed(seeds[i])
        counts[i] = 1
        rewards[i] = reward
    
    # UCB rounds
    for t in range(n_seeds, n_rounds):
        # Compute UCB values
        ucb_values = rewards / counts + np.sqrt(2 * np.log(t) / counts)
        
        # Select seed with highest UCB
        selected_idx = np.argmax(ucb_values)
        reward = evaluate_seed(seeds[selected_idx])
        
        # Update
        counts[selected_idx] += 1
        rewards[selected_idx] += reward
    
    # Return best seed
    best_idx = np.argmax(rewards / counts)
    return seeds[best_idx], rewards[best_idx] / counts[best_idx]
```

### 13.2 Bayesian Optimization

**Algorithm 13.2.1 (Bayesian Seed Optimization)**

```python
def bayesian_seed_optimization(seed_space, n_iterations=100):
    """
    Bayesian optimization for seed discovery.
    """
    # Initialize with random samples
    observed = []
    for _ in range(10):
        seed = sample_seed(seed_space)
        fitness = evaluate_fitness(seed)
        observed.append((seed, fitness))
    
    for i in range(n_iterations):
        # Fit surrogate model (Gaussian Process)
        X = np.array([s for s, f in observed])
        y = np.array([f for s, f in observed])
        gp = GaussianProcessRegressor()
        gp.fit(X, y)
        
        # Optimize acquisition function (Expected Improvement)
        def acquisition(x):
            mu, sigma = gp.predict(x.reshape(-1, 1), return_std=True)
            best_so_far = max(y)
            z = (mu - best_so_far) / sigma
            ei = (mu - best_so_far) * norm.cdf(z) + sigma * norm.pdf(z)
            return ei
        
        next_seed = optimize_acquisition(acquisition, seed_space)
        fitness = evaluate_fitness(next_seed)
        observed.append((next_seed, fitness))
    
    # Return best seed
    best = max(observed, key=lambda x: x[1])
    return best
```

### 13.3 Genetic Algorithms

**Algorithm 13.3.1 (Genetic Seed Optimization)**

```python
def genetic_seed_search(fitness_function, population_size=100, generations=50):
    """
    Genetic algorithm for seed optimization.
    """
    # Initialize population
    population = [random_seed() for _ in range(population_size)]
    
    for gen in range(generations):
        # Evaluate fitness
        fitness = [fitness_function(seed) for seed in population]
        
        # Selection (tournament)
        parents = []
        for _ in range(population_size // 2):
            tournament = random.sample(list(zip(population, fitness)), k=3)
            winner = max(tournament, key=lambda x: x[1])[0]
            parents.append(winner)
        
        # Crossover
        children = []
        for i in range(len(parents)):
            for j in range(i + 1, len(parents)):
                child = crossover_seeds(parents[i], parents[j])
                children.append(child)
        
        # Mutation
        children = [mutate_seed(c, mutation_rate=0.1) for c in children]
        
        # Survivor selection
        combined = list(zip(population, fitness)) + list(zip(children, [fitness_function(c) for c in children]))
        combined.sort(key=lambda x: x[1], reverse=True)
        population = [s for s, f in combined[:population_size]]
    
    # Return best
    fitness = [fitness_function(seed) for seed in population]
    best_idx = np.argmax(fitness)
    return population[best_idx], fitness[best_idx]

def crossover_seeds(parent1, parent2):
    """
    Crossover operation on seeds.
    """
    # Bit-level crossover
    bits1 = format(parent1, '064b')
    bits2 = format(parent2, '064b')
    
    crossover_point = random.randint(1, 63)
    child_bits = bits1[:crossover_point] + bits2[crossover_point:]
    
    return int(child_bits, 2)

def mutate_seed(seed, mutation_rate=0.1):
    """
    Bit-flip mutation on seed.
    """
    bits = list(format(seed, '064b'))
    
    for i in range(64):
        if random.random() < mutation_rate / 64:
            bits[i] = '1' if bits[i] == '0' else '0'
    
    return int(''.join(bits), 2)
```

### 13.4 Reinforcement Learning

**Definition 13.4.1 (Seed Selection MDP)**

- **State**: Current seed $S_t$ and history
- **Action**: Next seed $S_{t+1}$ (or modification)
- **Reward**: Fitness $\mathcal{F}(S_{t+1})$
- **Transition**: Deterministic (seed evaluation)

**Algorithm 13.4.1 (RL-Based Seed Search)**

```python
class SeedSearchAgent:
    """
    Reinforcement learning agent for seed search.
    """
    
    def __init__(self, seed_dim=64, hidden_dim=256):
        self.policy_network = nn.Sequential(
            nn.Linear(seed_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, seed_dim),
            nn.Sigmoid()  # Output probability for each bit
        )
        self.optimizer = optim.Adam(self.policy_network.parameters(), lr=0.001)
    
    def select_seed(self, current_seed=None):
        """
        Select next seed using policy network.
        """
        if current_seed is None:
            current_seed = torch.zeros(64)
        else:
            current_seed = torch.tensor([int(b) for b in format(current_seed, '064b')], dtype=torch.float32)
        
        probs = self.policy_network(current_seed)
        bits = (torch.rand(64) < probs).int()
        return int(''.join(map(str, bits.tolist())), 2)
    
    def update(self, seed, reward):
        """
        Update policy based on reward.
        """
        seed_tensor = torch.tensor([int(b) for b in format(seed, '064b')], dtype=torch.float32)
        probs = self.policy_network(seed_tensor)
        
        # Policy gradient loss
        log_prob = torch.sum(seed_tensor * torch.log(probs + 1e-10) + 
                            (1 - seed_tensor) * torch.log(1 - probs + 1e-10))
        loss = -log_prob * reward
        
        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()
```

3. **Best-of-k**: Select highest fitness
4. **Weighted**: Weight by historical fitness

```python
def construct_durable_ensemble(candidate_seeds, contexts, n_ensemble=5):
    """
    Construct a durable ensemble of seeds.
    """
    # Evaluate each seed across contexts
    scores = {}
    for seed in candidate_seeds:
        context_scores = []
        for context in contexts:
            score = evaluate_in_context(seed, context)
            context_scores.append(score)
        
        # Compute robustness score
        mean_score = np.mean(context_scores)
        variance = np.var(context_scores)
        robustness = mean_score - 0.5 * variance  # Risk-adjusted
        
        scores[seed] = {
            'mean': mean_score,
            'variance': variance,
            'robustness': robustness,
            'min': min(context_scores)
        }
    
    # Select diverse top performers
    sorted_seeds = sorted(scores.keys(), key=lambda s: scores[s]['robustness'], reverse=True)
    
    ensemble = [sorted_seeds[0]]
    for seed in sorted_seeds[1:]:
        # Check diversity
        min_diversity = min(compute_diversity(seed, e) for e in ensemble)
        if min_diversity > 0.3:  # Diversity threshold
            ensemble.append(seed)
            if len(ensemble) == n_ensemble:
                break
    
    return ensemble, [scores[s] for s in ensemble]
```

## Appendix A: Mathematical Proofs

### A.1 Proof of Theorem 1.2.1 (Seed Derivation Necessity)

**Complete Proof:**

Let $R = G(S^*, P^*)$ be the target repository. We show each requirement is necessary.

**Output Samples:** Without at least one output sample $R$, there is no information about $S^*$. The posterior $P(S | \emptyset) = P(S)$ provides no update.

**Model Specification:** Suppose we have model $M_1$ and $M_2$ with different mappings. Without knowing which generated $R$, we have:

$$P(S = S^* | R) = \sum_M P(M) P(S = S^* | R, M)$$

This is strictly less informative than $P(S = S^* | R, M^*)$.

**Prompt Knowledge:** Similarly, without prompt knowledge:

$$P(S = S^* | R) = \sum_P P(P) P(S = S^* | R, P)$$

**Determinism Guarantee:** If the model is non-deterministic, then $G(S, P)$ is a random variable. The mapping $S \to R$ is not well-defined, making unique recovery impossible.

$\square$

### A.2 Proof of Theorem 9.2.1 (Stable Attractor Existence)

**Complete Proof:**

Let $M$ be a deterministic model with prompt $P$. Define:

$$\mathcal{A} = \{S : G(S, P) \text{ is stable}\}$$

We show $\mathcal{A}$ is non-empty and forms an attractor.

**Non-emptiness:** Consider seeds that produce low-entropy outputs. For temperature $T = 0$, the model is deterministic, so all seeds produce stable outputs. Thus $\mathcal{A} \neq \emptyset$.

**Invariance:** If $S \in \mathcal{A}$ and $S'$ is sufficiently close such that the first $k$ tokens are identical, then by the autoregressive nature of language models:

$$G(S', P)_{:k} = G(S, P)_{:k}$$

This creates a region of similar outputs.

**Attraction:** Under gradient descent on the stability objective:

$$S_{t+1} = S_t - \eta \nabla_S \mathcal{L}_{stability}(S_t)$$

Seeds in the basin of $\mathcal{A}$ will converge to $\mathcal{A}$.

$\square$

```python
def full_stability_analysis(seed, prompt, model, n_runs=100):
    """
    Comprehensive stability analysis.
    
    Returns:
    - Output stability (exact match rate)
    - Semantic stability (embedding similarity)
    - Structural stability (AST similarity)
    - Token-level stability profile
    - Entropy analysis
    """
    results = {
        'output_stability': None,
        'semantic_stability': None,
        'structural_stability': None,
        'token_profile': None,
        'entropy': None
    }
    
    outputs = [model.generate(seed, prompt) for _ in range(n_runs)]
    
    # Output stability
    results['output_stability'] = exact_match_rate(outputs)
    
    # Semantic stability
    embeddings = [embed(o) for o in outputs]
    pairwise_sim = [cosine_sim(e1, e2) for e1, e2 in combinations(embeddings, 2)]
    results['semantic_stability'] = np.mean(pairwise_sim)
    
    # Structural stability
    asts = [parse_ast(o) for o in outputs]
    ast_sims = [ast_similarity(a1, a2) for a1, a2 in combinations(asts, 2)]
    results['structural_stability'] = np.mean(ast_sims)
    
    # Token profile
    results['token_profile'] = analyze_token_positions(outputs)
    
    # Entropy
    results['entropy'] = compute_output_entropy(outputs)
    
    return results
```

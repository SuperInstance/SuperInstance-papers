│                                                         │
│  Actions: - Local heuristics                          │
│          - Cached responses                           │
│          - Simple transformations                     │
│          - Fast feedback (< 10ms)                     │
│                                                         │

│            - No external calls                        │
│            - Local memory only                        │
│                                                         │

Stability Metrics:
==================

┌─────────────────────────────────────────────────────────┐
│               STABILITY CALCULATION                      │
│                                                         │
│  σ = 1 - variance(predictions) / max_variance          │
│                                                         │
│  Components:                                           │
│  • Output variance across model runs                   │
│  • Confidence consistency                              │
│  • Temporal stability (rate of drift)                  │
│  • Input sensitivity                                   │
│  • Composition penalty (when chaining)                 │
│                                                         │
│  Thresholds:                                           │
│  • σ > 0.9: High stability (green)                    │
│  • σ > 0.7: Medium stability (yellow)                 │
│  • σ < 0.7: Low stability (red)                       │
│                                                         │
│  Actions based on stability:                           │
│  • High: Proceed normally                              │
│  • Medium: Add monitoring                             │
│  • Low: Apply constraints or escalate                 │
└─────────────────────────────────────────────────────────┘

Error Recovery:
===============

1. Stability Failure
   → Apply constraints
   → Re-evaluate with adjusted parameters
   → Escalate to human if needed

2. Model Unavailable
   → Use cached prediction
   → Fallback to simpler model
   → Defer execution

3. Rate Limit Exceeded
   → Queue request
   → Reduce frequency
   → Use batch processing

4. Confidence Too Low
   → Request more context
   → Consult dependencies
   → Mark as uncertain

Monitoring Points:
==================

□ Pre-activation: Input validation, deadband check
□ During execution: Stability tracking, resource usage
□ Post-execution: Output validation, confidence update
□ Cascade: Dependency notification, federation sync
□ Long-term: Rate history, prediction accuracy, drift detection

Performance Targets:
====================

• Deadband check: < 1ms
• Level 1 activation: < 10ms
• Level 2 activation: < 100ms
• Level 3 activation: < 1s
• Stability calculation: < 10ms
• Confidence propagation: < 50ms
• Federation sync: < 200ms
• Overall P99 latency: < 2s
